# AIGC Showcase 设计交付需求

这份文档给设计协作者使用。当前网站 MVP 已上线，下一步需要补齐视觉设计、作品素材和页面内容，使网站从“可运行版本”进入“正式展示版本”。

## 1. 项目定位

网站用途：AIGC 创作团队作品展示官网。

当前页面结构：

- 首页：Hero 视频背景 + 团队主标题 + 作品列表
- About：团队介绍、工具标签、商业合作入口
- 作品详情页：单个作品的视频、图片、说明、标签、复盘链接、主创信息

当前技术栈：

- Next.js 14
- Tailwind CSS
- TypeScript
- Vercel 部署

## 2. 设计目标

希望视觉方向偏“作品集 / 创作厂牌官网”，而不是普通公司官网。

关键词：

- AIGC
- 影像感
- 实验性
- 高级、克制、有冲击力
- 风格不限，突破极限

当前站点已有黑底 + 荧光黄点缀，可以保留，也可以给出新的完整视觉方向。设计时请优先考虑作品展示的质感，不要做成营销落地页。

## 3. 需要设计的页面

### 3.1 首页

需要设计：

- 顶部导航栏
- 首屏 Hero 区
- Hero 标题、口号、视频遮罩方式
- 向下滚动提示
- 作品列表区域
- 空状态或作品不足时的展示方式
- 移动端首页布局

首页首屏当前会播放一条背景视频。请设计时考虑视频画面较暗、较亮、对比强烈三种情况，避免文字看不清。

### 3.2 作品列表卡片

每个作品卡片需要包含：

- 封面图
- 作品类型
- 标题
- 一句话介绍
- 标签
- 时长，可选

请提供：

- 默认状态
- Hover 状态
- 移动端卡片状态
- 没有封面图时的占位状态

### 3.3 作品详情页

详情页需要设计：

- 顶部返回 / 导航方式
- 主视频或主图区域
- 标题区
- 项目简介
- 图片组 / 分镜图 / 海报图展示方式
- 创作工具或标签
- 主创名单
- 复盘文章入口

详情页应适合展示以下类型：

- AI 短片
- 系列短剧
- 图像作品
- 商业短视频

### 3.4 About 页面

需要设计：

- 团队介绍版式
- 核心 slogan
- 工具标签区域
- 商业合作联系入口
- 页脚信息

About 页面文字可以先用占位文案，后续我会替换成正式中文内容。

## 4. 需要提供的设计资产

### 4.1 品牌基础

请提供：

- Logo 或文字标识，SVG 优先
- 黑底版本
- 白底版本，如有必要
- Favicon，建议 `512x512 PNG` 和 `SVG`
- 主色、辅助色、文字色、背景色
- 字体建议，中英文字体都需要

如果暂时没有完整 Logo，也可以先提供文字标识方案。

### 4.2 图片和视频素材

请提供以下素材：

- 首页 Hero 视频，`mp4`，推荐 `1920x1080` 或更高，横版 16:9
- Hero 视频首帧备用图，`jpg` 或 `webp`，推荐 `1920x1080`
- 每个作品的封面图，推荐 `1920x1080`，横版 16:9
- 作品详情页图片，推荐横版 16:9 或竖版 4:5，但同一个作品内尽量统一
- 如有海报，可额外提供 `1080x1920` 竖版

视频建议：

- 编码：H.264
- 格式：`.mp4`
- 单个 Hero 视频建议小于 `20MB`
- 单个作品预览视频建议小于 `80MB`
- 不要直接交 `.mov` 作为线上素材，除非同时提供压缩后的 `.mp4`

### 4.3 图标和装饰元素

如设计中使用图标或装饰元素，请提供：

- SVG 原文件
- 使用说明
- 是否允许前端用 CSS / lucide icon 近似替代

不建议大量使用纯装饰素材。这个站点重点是作品本身。

## 5. 作品资料交付格式

每个作品请按下面格式提供一个资料包。

文件夹命名：

```text
works/作品英文slug/
```

示例：

```text
works/ice-city/
```

每个作品资料包包含：

```text
works/ice-city/
  cover.webp
  preview.mp4
  image-01.webp
  image-02.webp
  image-03.webp
  info.md
```

`info.md` 请使用这个格式：

```markdown
# 作品标题

slug: ice-city
type: short-film
year: 2026
duration: 02:14
tagline: 一句话介绍，控制在 20-40 字
tags: AIGC, 短片, 科幻, Midjourney, Seedance
postmortemUrl:

## Description

这里写作品简介，建议 100-300 字。

## Credits

- 导演：
- 视觉：
- 剪辑：
- 音乐：
- 工具：
```

`type` 只能使用以下四种之一：

```text
short-film
series
image
short-video
```

## 6. 文件命名规则

请使用英文小写、数字和短横线，避免中文、空格和特殊符号。

推荐：

```text
hero-bg.mp4
hero-poster.webp
cover.webp
preview.mp4
image-01.webp
image-02.webp
logo.svg
favicon.png
```

不推荐：

```text
最终版.mp4
首页 视频 1.mov
作品封面(新)new.jpg
```

## 7. 设计稿交付格式

优先交付：

- Figma 链接
- 关键页面 PNG 导出
- 素材包 zip
- 字体名称和下载 / 授权说明

Figma 中请至少包含：

- Desktop 首页，建议宽度 `1440`
- Mobile 首页，建议宽度 `390`
- Desktop 作品详情页，建议宽度 `1440`
- Mobile 作品详情页，建议宽度 `390`
- About 页面
- 颜色、字体、间距说明
- 组件状态，如按钮、卡片 hover、标签、导航

PNG 导出建议：

```text
design-preview/
  home-desktop.png
  home-mobile.png
  work-detail-desktop.png
  work-detail-mobile.png
  about-desktop.png
```

## 8. 前端接入需要的信息

为了让我快速接入，请同时说明：

- 哪些设计必须严格还原
- 哪些地方可以由前端按响应式规则自由调整
- 是否有固定品牌色
- 是否有必须使用的字体
- 是否有动效要求
- Hero 视频是否需要自动播放、循环、静音
- 作品卡片点击后进入详情页还是外链
- 商务合作邮箱或联系方式

如果时间紧，先交这一批就够我继续开发：

- 首页 Desktop + Mobile 设计稿

- 作品详情页 Desktop + Mobile 设计稿
- Logo 或文字标识

- 3 个作品资料包，每个包含封面、视频或图片、`info.md`
